# WebAgentFlow Validation Site

This repository contains a standalone product-like validation site for WebAgentFlow.

It is not the WebAgentFlow product UI.
It is not an internal deterministic fixture site.
It is not a compatibility mirror of `apps/product-test-site`.

The first validation surface is an inventory management page:

```text
http://127.0.0.1:5177/inventory
```

This repository migrates product validation scenario intent from the historical
`apps/product-test-site`, but it does not preserve legacy routes such as
`/items`, `/workspace-login`, or `/workspace-home`.

## Repository Role

`WebAgentFlow-Validation-Site` is an external product-level validation target.
It gives WebAgentFlow a realistic business page for black-box browser operation
checks without colocating the site implementation with WebAgentFlow runtime code,
prompts, or eval runners.

The first scenario is **Inventory Validation Site**. It supports safe inventory
operations:

- create an inventory item
- search inventory items
- edit an inventory item

## Boundary

The WebAgentFlow main repository may know:

- target URL
- high-level user intent
- expected visible behavior
- forbidden behavior
- result artifacts

The WebAgentFlow main repository should not know:

- page source code
- component names
- `data-testid` values
- DOM selectors
- seed copy used by this repository
- internal state shape
- implementation-specific business logic

## Development

```bash
pnpm install
pnpm dev
```

Default local target:

```text
http://127.0.0.1:5177/inventory
```

LAN development target:

```bash
pnpm dev:lan
```

## Build

```bash
pnpm build
```

`pnpm-lock.yaml` is generated by `pnpm install`; do not hand-write it.
